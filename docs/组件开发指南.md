# RiskFlow 组件开发指南

## 1. 组件概述

在 RiskFlow 中，组件（Component）是 LiteFlow 流程编排的基本单元。每个组件代表一个独立的风控检测或处理逻辑。

### 1.1 组件分类

| 分类 | 说明 | 示例 |
|------|------|------|
| **基础检测组件** | 基础的黑白名单检测、IP检测等 | IpBlacklistCheckComponent |
| **规则执行组件** | 执行业务规则，计算风险评分 | RuleExecuteComponent |
| **AI 分析组件** | 使用 AI 进行智能分析 | LlmRiskAnalyzeComponent |
| **决策组件** | 根据评分做出最终决策 | DecisionJudgeComponent |
| **后处理组件** | 日志记录、通知等后处理 | LogRecordComponent |

### 1.2 组件生命周期

```
1. 组件注册 (@LiteflowComponent)
2. 流程加载 (risk-decision.xml)
3. 流程执行 (FlowExecutor.execute)
4. 组件执行 (process() 方法)
5. 结果返回 (通过 Context 传递)
```

## 2. 组件开发步骤

### 2.1 创建组件类

继承 `NodeComponent` 并使用 `@LiteflowComponent` 注解：

```java
package org.example.component.base;

import com.yomahub.liteflow.annotation.LiteflowComponent;
import com.yomahub.liteflow.core.NodeComponent;
import org.example.context.RiskFlowContext;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

@LiteflowComponent("myComponent")  // 节点ID，在流程中引用
public class MyComponent extends NodeComponent {
    
    private static final Logger log = LoggerFactory.getLogger(MyComponent.class);
    
    @Override
    public void process() {
        // 1. 获取上下文
        RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
        
        // 2. 执行业务逻辑
        // ...
        
        // 3. 更新上下文
        // ...
        
        log.info("[MyComponent] 执行完成");
    }
    
    @Override
    public boolean isAccess() {
        // 控制组件是否执行
        return true;
    }
}
```

### 2.2 在流程中引用

在 `risk-decision.xml` 中引用组件：

```xml
<flow>
    <chain name="riskDecisionFlow">
        THEN(
            loadContext,
            myComponent,  <!-- 使用组件ID -->
            decisionJudge
        );
    </chain>
</flow>
```

### 2.3 注入依赖服务

如果组件需要依赖其他服务，使用构造函数注入：

```java
@LiteflowComponent("myComponent")
public class MyComponent extends NodeComponent {
    
    private final MyService myService;
    
    public MyComponent(MyService myService) {
        this.myService = myService;
    }
    
    @Override
    public void process() {
        RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
        
        // 使用注入的服务
        boolean result = myService.check(context.getUserIp());
        
        if (result) {
            context.addRuleScore("my_rule", 50);
        }
    }
}
```

## 3. 组件开发最佳实践

### 3.1 单一职责原则

每个组件只做一件事，保持简单和可复用。

**❌ 不好的做法**：
```java
@LiteflowComponent("checkAll")
public class CheckAllComponent extends NodeComponent {
    @Override
    public void process() {
        // 检查IP黑名单
        // 检查设备黑名单
        // 检查用户黑名单
        // 执行规则
        // 做决策
        // ... 太多职责
    }
}
```

**✅ 好的做法**：
```java
@LiteflowComponent("ipBlacklistCheck")
public class IpBlacklistCheckComponent extends NodeComponent {
    @Override
    public void process() {
        // 只检查IP黑名单
    }
}

@LiteflowComponent("deviceBlacklistCheck")
public class DeviceBlacklistCheckComponent extends NodeComponent {
    @Override
    public void process() {
        // 只检查设备黑名单
    }
}
```

### 3.2 使用 Context 传递数据

组件之间通过 `RiskFlowContext` 传递数据，不要使用静态变量或其他全局状态。

**✅ 正确做法**：
```java
@Override
public void process() {
    RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
    
    // 读取数据
    String userIp = context.getUserIp();
    
    // 写入数据
    context.addRuleScore("ip_check", 50);
    context.setBaseCheckResult("ip_blacklist", true);
}
```

### 3.3 异常处理

组件内部应该捕获异常，避免影响整个流程。

```java
@Override
public void process() {
    RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
    
    try {
        // 业务逻辑
        boolean isBlacklisted = blacklistService.isIpBlacklisted(context.getUserIp());
        
        if (isBlacklisted) {
            context.addRuleScore("ip_blacklist", 50);
            log.warn("[IpBlacklistCheck] IP命中黑名单: {}", context.getUserIp());
        }
    } catch (Exception e) {
        log.error("[IpBlacklistCheck] 检测失败: {}", e.getMessage(), e);
        // 可以选择：
        // 1. 继续执行（容错）
        // 2. 设置默认值
        // 3. 抛出异常（中断流程）
    }
}
```

### 3.4 日志记录

使用结构化日志，便于追踪和分析。

```java
private static final Logger log = LoggerFactory.getLogger(MyComponent.class);

@Override
public void process() {
    RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
    
    log.info("[MyComponent] 开始执行, eventId={}, userId={}", 
            context.getEventId(), context.getUserId());
    
    // 业务逻辑
    
    log.info("[MyComponent] 执行完成, score={}", context.getTotalRiskScore());
}
```

### 3.5 性能优化

- 避免重复计算，使用缓存
- 避免阻塞操作，使用异步
- 避免大对象创建，复用对象

```java
@LiteflowComponent("ipGeoLocation")
public class IpGeoLocationComponent extends NodeComponent {
    
    private final GeoIpService geoIpService;
    private final Cache<String, GeoLocation> cache;
    
    @Override
    public void process() {
        RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
        String ip = context.getUserIp();
        
        // 使用缓存
        GeoLocation location = cache.get(ip, () -> geoIpService.lookup(ip));
        
        context.setFeature("country", location.getCountry());
        context.setFeature("city", location.getCity());
    }
}
```

## 4. 常见组件模式

### 4.1 检测型组件

检测某个条件是否满足，命中则加分。

```java
@LiteflowComponent("ipBlacklistCheck")
public class IpBlacklistCheckComponent extends NodeComponent {
    
    private final BlacklistService blacklistService;
    
    public IpBlacklistCheckComponent(BlacklistService blacklistService) {
        this.blacklistService = blacklistService;
    }
    
    @Override
    public void process() {
        RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
        String userIp = context.getUserIp();
        
        if (userIp == null || userIp.isEmpty()) {
            return;
        }
        
        boolean isBlacklisted = blacklistService.isIpBlacklisted(userIp);
        
        if (isBlacklisted) {
            context.addRuleScore("ip_blacklist", 50);
            context.setBaseCheckResult("ip_blacklist", true);
            log.warn("[IpBlacklistCheck] IP命中黑名单: {}", userIp);
        } else {
            context.setBaseCheckResult("ip_blacklist", false);
        }
    }
}
```

### 4.2 规则型组件

执行一组规则，累加评分。

```java
@LiteflowComponent("ruleExecute")
public class RuleExecuteComponent extends NodeComponent {
    
    private final RuleService ruleService;
    
    public RuleExecuteComponent(RuleService ruleService) {
        this.ruleService = ruleService;
    }
    
    @Override
    public void process() {
        RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
        
        // 获取适用的规则
        List<Rule> rules = ruleService.getRulesByEventType(context.getEventType());
        
        // 执行规则
        List<Rule> matchedRules = ruleService.executeRules(context);
        
        log.info("[RuleExecute] 执行了 {} 条规则，命中 {} 条", 
                rules.size(), matchedRules.size());
    }
}
```

### 4.3 分析型组件

对数据进行分析，生成分析结果。

```java
@LiteflowComponent("llmRiskAnalyze")
public class LlmRiskAnalyzeComponent extends NodeComponent {
    
    private final LlmService llmService;
    
    public LlmRiskAnalyzeComponent(LlmService llmService) {
        this.llmService = llmService;
    }
    
    @Override
    public void process() {
        RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
        
        // 构建 Prompt
        String prompt = buildPrompt(context);
        
        // 调用 LLM
        String analysis = llmService.analyze(prompt);
        
        // 保存分析结果
        context.setAiAnalysisResult(analysis);
        
        log.info("[LlmRiskAnalyze] AI分析完成");
    }
    
    private String buildPrompt(RiskFlowContext context) {
        return String.format(
            "分析以下风控数据：\n" +
            "用户ID: %s\n" +
            "事件类型: %s\n" +
            "风险评分: %d\n" +
            "特征数据: %s\n",
            context.getUserId(),
            context.getEventType(),
            context.getTotalRiskScore(),
            context.getFeatures()
        );
    }
}
```

### 4.4 决策型组件

根据评分做出最终决策。

```java
@LiteflowComponent("decisionJudge")
public class DecisionJudgeComponent extends NodeComponent {
    
    private static final int REJECT_THRESHOLD = 80;
    private static final int REVIEW_THRESHOLD = 50;
    private static final int CHALLENGE_THRESHOLD = 30;
    
    @Override
    public void process() {
        RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
        int totalScore = context.getTotalRiskScore();
        
        if (totalScore >= REJECT_THRESHOLD) {
            context.setResult(RiskFlowContext.DecisionResult.REJECT);
            context.setResultMessage("高风险操作，系统直接拒绝");
        } else if (totalScore >= REVIEW_THRESHOLD) {
            context.setResult(RiskFlowContext.DecisionResult.REVIEW);
            context.setResultMessage("存在一定风险，建议人工审核");
        } else if (totalScore >= CHALLENGE_THRESHOLD) {
            context.setResult(RiskFlowContext.DecisionResult.CHALLENGE);
            context.setResultMessage("需要通过验证码验证");
        } else {
            context.setResult(RiskFlowContext.DecisionResult.ACCEPT);
            context.setResultMessage("风险可控，审核通过");
        }
        
        context.setIsHighRisk(totalScore >= REVIEW_THRESHOLD);
        context.setDecisionTime(LocalDateTime.now());
        
        log.info("[DecisionJudge] 决策完成，总评分={}, 决策结果={}", 
                totalScore, context.getResult());
    }
}
```

## 5. 高级特性

### 5.1 条件执行

使用 `isAccess()` 方法控制组件是否执行。

```java
@LiteflowComponent("llmRiskAnalyze")
public class LlmRiskAnalyzeComponent extends NodeComponent {
    
    @Override
    public void process() {
        // AI 分析逻辑
    }
    
    @Override
    public boolean isAccess() {
        RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
        // 只有高风险时才执行 AI 分析
        return context.getTotalRiskScore() >= 50;
    }
}
```

### 5.2 条件组件

创建专门的条件判断组件，用于流程分支。

```java
@LiteflowComponent("isHighRisk")
public class IsHighRiskComponent extends NodeBooleanComponent {
    
    @Override
    public boolean processBoolean() {
        RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
        return context.getTotalRiskScore() >= 50;
    }
}
```

在流程中使用：

```xml
<chain name="riskDecisionFlow">
    THEN(
        loadContext,
        ruleExecute,
        IF(isHighRisk,
            THEN(llmRiskAnalyze),
            THEN(skipAiAnalyze)
        ),
        decisionJudge
    );
</chain>
```

### 5.3 并行执行

使用 `WHEN` 实现并行执行。

```xml
<chain name="parallelBaseCheck">
    WHEN(
        ipBlacklistCheck,
        deviceBlacklistCheck,
        userBlacklistCheck
    );
</chain>
```

### 5.4 异步执行

使用 `@Async` 注解实现异步执行。

```java
@LiteflowComponent("asyncNotify")
public class AsyncNotifyComponent extends NodeComponent {
    
    private final NotificationService notificationService;
    
    @Override
    public void process() {
        RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
        
        // 异步发送通知
        notificationService.sendAsync(context);
    }
}
```

## 6. 测试

### 6.1 单元测试

```java
@SpringBootTest
class IpBlacklistCheckComponentTest {
    
    @Autowired
    private FlowExecutor flowExecutor;
    
    @MockBean
    private BlacklistService blacklistService;
    
    @Test
    void testIpBlacklistHit() {
        // Mock 黑名单服务
        when(blacklistService.isIpBlacklisted("192.168.1.100"))
            .thenReturn(true);
        
        // 准备上下文
        RiskFlowContext context = new RiskFlowContext();
        context.setEventId("test-001");
        context.setUserIp("192.168.1.100");
        
        // 执行流程
        flowExecutor.execute("riskDecisionFlow", context);
        
        // 验证结果
        assertTrue(context.getBaseCheckResult("ip_blacklist"));
        assertEquals(50, context.getRuleScores().get("ip_blacklist"));
    }
}
```

### 6.2 集成测试

```java

@AutoConfigureTestDatabase
```

## 7. 常见问题

### 7.1 组件没有被执行

**原因**：
- 组件没有注册（缺少 `@LiteflowComponent` 注解）
- 流程定义中没有引用该组件
- `isAccess()` 返回 false

**解决**：
- 检查注解是否正确
- 检查流程定义文件
- 检查 `isAccess()` 逻辑

### 7.2 获取不到 Context

**原因**：
- Context 类型不匹配
- Context 没有正确传递

**解决**：
```java
// 正确的方式
RiskFlowContext context = this.getContextBean(RiskFlowContext.class);

// 错误的方式
RiskFlowContext context = (RiskFlowContext) this.getContext();
```

### 7.3 依赖注入失败

**原因**：
- 服务没有被 Spring 管理
- 循环依赖

**解决**：
- 确保服务类有 `@Service` 或 `@Component` 注解
- 使用构造函数注入，避免循环依赖

### 7.4 并发问题

**原因**：
- 组件使用了实例变量
- 共享可变状态

**解决**：
```java
// ❌ 错误：使用实例变量
@LiteflowComponent("myComponent")
public class MyComponent extends NodeComponent {
    private int counter = 0;  // 并发不安全
    
    @Override
    public void process() {
        counter++;  // 多线程问题
    }
}

// ✅ 正确：使用 Context 传递数据
@LiteflowComponent("myComponent")
public class MyComponent extends NodeComponent {
    @Override
    public void process() {
        RiskFlowContext context = this.getContextBean(RiskFlowContext.class);
        // 使用 Context 中的数据
    }
}
```

## 8. 组件开发检查清单

开发新组件时，请检查以下事项：

- [ ] 组件类继承 `NodeComponent`
- [ ] 使用 `@LiteflowComponent` 注解，指定唯一的节点ID
- [ ] 实现 `process()` 方法
- [ ] 通过 `getContextBean()` 获取上下文
- [ ] 使用 Context 传递数据，不使用实例变量
- [ ] 添加适当的日志记录
- [ ] 处理异常情况
- [ ] 编写单元测试
- [ ] 在流程定义中引用该组件
- [ ] 更新文档

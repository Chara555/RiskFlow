# RiskFlow 数据模型设计

## 1. 数据库选型

**PostgreSQL 14+**

选型理由：
- 原生支持 JSONB 类型，适合存储动态特征数据
- 高性能、高并发
- 支持复杂查询和索引优化
- 开源免费

---

## 2. 核心表设计

### 2.1 workflow (流程定义表)

存储风控流程的定义和配置。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGSERIAL | PRIMARY KEY | 主键 |
| name | VARCHAR(100) | NOT NULL | 流程名称 |
| code | VARCHAR(50) | NOT NULL, UNIQUE | 流程编码（唯一标识） |
| description | VARCHAR(500) | | 流程描述 |
| version | INTEGER | NOT NULL, DEFAULT 1 | 版本号 |
| status | VARCHAR(20) | DEFAULT 'DRAFT' | 状态：DRAFT/PUBLISHED/ARCHIVED |
| flow_data | JSONB | | 流程配置数据（JSON格式） |
| created_by | VARCHAR(50) | | 创建人 |
| created_at | TIMESTAMP | NOT NULL | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL | 更新时间 |

**索引**：
```sql
CREATE INDEX idx_workflow_code ON workflow(code);
CREATE INDEX idx_workflow_status ON workflow(status);
```

**示例数据**：
```json
{
  "id": 1,
  "name": "通用风控决策流程",
  "code": "risk-decision-flow",
  "version": 1,
  "status": "PUBLISHED",
  "flow_data": {
    "nodes": [
      {"id": "loadContext", "type": "base", "name": "加载上下文"},
      {"id": "ipBlacklistCheck", "type": "check", "name": "IP黑名单检测"},
      {"id": "ruleExecute", "type": "rule", "name": "规则执行"},
      {"id": "llmRiskAnalyze", "type": "ai", "name": "AI分析"},
      {"id": "decisionJudge", "type": "decision", "name": "决策判定"}
    ],
    "edges": [
      {"from": "loadContext", "to": "ipBlacklistCheck"},
      {"from": "ipBlacklistCheck", "to": "ruleExecute"}
    ]
  }
}
```

### 2.2 rule_config (规则配置表)

存储风控规则的配置信息。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGSERIAL | PRIMARY KEY | 主键 |
| code | VARCHAR(50) | NOT NULL, UNIQUE | 规则编码 |
| name | VARCHAR(100) | NOT NULL | 规则名称 |
| type | VARCHAR(30) | | 规则类型：FREQUENCY/VELOCITY/BLACKLIST/CUSTOM |
| expression | TEXT | | 规则表达式 |
| score | INTEGER | NOT NULL, DEFAULT 0 | 风险评分 |
| enabled | BOOLEAN | NOT NULL, DEFAULT TRUE | 是否启用 |
| priority | INTEGER | NOT NULL, DEFAULT 0 | 优先级（数字越大优先级越高） |
| created_at | TIMESTAMP | NOT NULL | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL | 更新时间 |

**索引**：
```sql
CREATE INDEX idx_rule_code ON rule_config(code);
CREATE INDEX idx_rule_enabled ON rule_config(enabled);
CREATE INDEX idx_rule_priority ON rule_config(priority DESC);
```

### 2.4 decision_log (决策日志表)

存储每次风控决策的详细日志。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGSERIAL | PRIMARY KEY | 主键 |
| event_id | VARCHAR(100) | NOT NULL | 事件ID（唯一标识一次决策） |
| workflow_id | BIGINT | | 流程ID（外键） |
| user_id | VARCHAR(100) | | 用户ID |

| request_data | JSONB | | 请求数据（JSON格式） |
| risk_score | INTEGER | | 风险评分 |

| decision_msg | VARCHAR(500) | | 决策消息 |

CREATE INDEX idx_decision_event_id ON decision_log(event_id);

CREATE INDEX idx_decision_created_at ON decision_log(created_at DESC);

## 4. 数据库初始化脚本

### 4.1 创建数据库

```sql
CREATE DATABASE riskflow
    WITH 
    OWNER = postgres
    ENCODING = 'UTF8'
    LC_COLLATE = 'en_US.UTF-8'
    LC_CTYPE = 'en_US.UTF-8'
    TABLESPACE = pg_default
    CONNECTION LIMIT = -1;
```

### 4.2 创建表结构

```sql
-- 流程定义表
CREATE TABLE workflow (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    code VARCHAR(50) NOT NULL UNIQUE,
    description VARCHAR(500),
    version INTEGER NOT NULL DEFAULT 1,
    status VARCHAR(20) DEFAULT 'DRAFT',
    flow_data JSONB,
    created_by VARCHAR(50),
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_workflow_code ON workflow(code);
CREATE INDEX idx_workflow_status ON workflow(status);

-- 规则配置表
CREATE TABLE rule_config (
    id BIGSERIAL PRIMARY KEY,
    code VARCHAR(50) NOT NULL UNIQUE,
    name VARCHAR(100) NOT NULL,
    type VARCHAR(30),
    expression TEXT,
    score INTEGER NOT NULL DEFAULT 0,
    enabled BOOLEAN NOT NULL DEFAULT TRUE,
    priority INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_rule_code ON rule_config(code);
CREATE INDEX idx_rule_enabled ON rule_config(enabled);
CREATE INDEX idx_rule_priority ON rule_config(priority DESC);

-- 黑名单表
CREATE TABLE blacklist (
    id BIGSERIAL PRIMARY KEY,
    type VARCHAR(20) NOT NULL,
    value VARCHAR(200) NOT NULL,
    reason VARCHAR(500),
    source VARCHAR(50),
    expire_time TIMESTAMP,
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_blacklist_type_value ON blacklist(type, value);
CREATE INDEX idx_blacklist_expire_time ON blacklist(expire_time);

-- 决策日志表
CREATE TABLE decision_log (
    id BIGSERIAL PRIMARY KEY,
    event_id VARCHAR(100) NOT NULL,
    workflow_id BIGINT,
    user_id VARCHAR(100),
    event_type VARCHAR(50),
    request_data JSONB,
    risk_score INTEGER,
    decision VARCHAR(20),
    decision_msg VARCHAR(500),
    node_results JSONB,
    execution_time INTEGER,
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_decision_event_id ON decision_log(event_id);
CREATE INDEX idx_decision_user_id ON decision_log(user_id);
CREATE INDEX idx_decision_event_type ON decision_log(event_type);
CREATE INDEX idx_decision_created_at ON decision_log(created_at DESC);
CREATE INDEX idx_decision_risk_score ON decision_log(risk_score DESC);
```

## 5. 数据查询示例

### 5.1 查询用户的决策历史

```sql
SELECT 
    event_id,
    event_type,
    risk_score,
    decision,
    decision_msg,
    execution_time,
    created_at
FROM decision_log
WHERE user_id = 'user123'
ORDER BY created_at DESC
LIMIT 10;
```

### 5.2 统计各决策结果的数量

```sql
SELECT 
    decision,
    COUNT(*) as count,
    AVG(risk_score) as avg_score,
    AVG(execution_time) as avg_time
FROM decision_log
WHERE created_at >= NOW() - INTERVAL '7 days'
GROUP BY decision
ORDER BY count DESC;
```

### 5.3 查询高风险用户

```sql
SELECT 
    user_id,
    COUNT(*) as total_events,
    SUM(CASE WHEN decision = 'REJECT' THEN 1 ELSE 0 END) as reject_count,
    AVG(risk_score) as avg_risk_score
FROM decision_log
WHERE created_at >= NOW() - INTERVAL '30 days'
GROUP BY user_id
HAVING AVG(risk_score) > 50
ORDER BY avg_risk_score DESC
LIMIT 20;
```

### 5.4 查询规则命中率

```sql
SELECT 
    jsonb_array_elements_text(node_results->'ruleExecute'->'matchedRules') as rule_code,
    COUNT(*) as hit_count
FROM decision_log
WHERE created_at >= NOW() - INTERVAL '7 days'
    AND node_results->'ruleExecute'->'matchedRules' IS NOT NULL
GROUP BY rule_code
ORDER BY hit_count DESC;
```

## 6. 数据归档策略

### 6.1 归档规则

- **decision_log**：保留最近 90 天的数据，超过 90 天的数据归档到历史表
- **rule_execution_log**：保留最近 30 天的数据
- **ai_analysis_log**：保留最近 30 天的数据

### 6.2 归档脚本示例

```sql
-- 创建归档表
CREATE TABLE decision_log_archive (LIKE decision_log INCLUDING ALL);

-- 归档数据
INSERT INTO decision_log_archive
SELECT * FROM decision_log
WHERE created_at < NOW() - INTERVAL '90 days';

-- 删除已归档数据
DELETE FROM decision_log
WHERE created_at < NOW() - INTERVAL '90 days';
```

## 7. 性能优化建议

### 7.1 分区表

对于 `decision_log` 表，建议按时间分区：

```sql
CREATE TABLE decision_log (
    id BIGSERIAL,
    event_id VARCHAR(100) NOT NULL,
    -- ... 其他字段
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
) PARTITION BY RANGE (created_at);

-- 创建分区
CREATE TABLE decision_log_2026_03 PARTITION OF decision_log
    FOR VALUES FROM ('2026-03-01') TO ('2026-04-01');

CREATE TABLE decision_log_2026_04 PARTITION OF decision_log
    FOR VALUES FROM ('2026-04-01') TO ('2026-05-01');
```

### 7.2 物化视图

创建物化视图用于统计分析：

```sql
CREATE MATERIALIZED VIEW mv_daily_decision_stats AS
SELECT 
    DATE(created_at) as date,
    event_type,
    decision,
    COUNT(*) as count,
    AVG(risk_score) as avg_score,
    AVG(execution_time) as avg_time
FROM decision_log
GROUP BY DATE(created_at), event_type, decision;

-- 创建索引
CREATE INDEX idx_mv_daily_stats_date ON mv_daily_decision_stats(date DESC);

-- 定时刷新
REFRESH MATERIALIZED VIEW mv_daily_decision_stats;
```

### 7.3 JSONB 索引

对于 JSONB 字段，创建 GIN 索引：

```sql
CREATE INDEX idx_decision_request_data ON decision_log USING GIN (request_data);
CREATE INDEX idx_decision_node_results ON decision_log USING GIN (node_results);
```

## 8. 数据安全

### 8.1 敏感数据加密

对于敏感字段（如手机号、邮箱），建议使用 PostgreSQL 的 pgcrypto 扩展进行加密：

```sql
CREATE EXTENSION pgcrypto;

-- 加密存储
INSERT INTO blacklist (type, value, reason, source)
VALUES ('PHONE', pgp_sym_encrypt('13800138000', 'encryption_key'), '垃圾注册', 'AUTO');

-- 解密查询
SELECT pgp_sym_decrypt(value::bytea, 'encryption_key') as phone
FROM blacklist
WHERE type = 'PHONE';
```

### 8.2 数据脱敏

在非生产环境，对敏感数据进行脱敏：

```sql
UPDATE decision_log
SET request_data = jsonb_set(
    request_data,
    '{userId}',
    to_jsonb('user_' || SUBSTRING(MD5(request_data->>'userId'), 1, 8))
)
WHERE created_at < NOW() - INTERVAL '30 days';
```

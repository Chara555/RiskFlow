# RiskFlow 后续开发规划

## 1. 开发优先级分级

### 🔴 P0 - 核心功能完善（1-2周）

必须完成的核心功能，直接影响系统可用性。

### 🟡 P1 - 重要功能增强（2-4周）

重要但非紧急的功能，提升系统能力。

### 🟢 P2 - 优化和扩展（4-8周）

优化性能、用户体验的功能。

### 🔵 P3 - 长期规划（8周+）

长期愿景，可分阶段实施。

## 2. P0 - 核心功能完善

### 2.1 完善基础检测组件

**目标**：补充常用的基础检测能力

#### 2.1.1 IP 相关组件

- [ ] **IpWhitelistCheckComponent** - IP 白名单检测
  - 白名单命中直接通过，不累加风险分
  - 支持 CIDR 格式（如 192.168.0.0/16）
  - 支持 IPv6

- [ ] **IpGeoLocationComponent** - IP 地理位置检测
  - 集成 GeoIP2 或 IP2Location
  - 检测 IP 归属地
  - 识别代理/VPN/Tor 节点
  - 检测地理位置异常（如用户常驻地突变）

#### 2.1.2 设备相关组件

- [ ] **DeviceFingerprintComponent** - 设备指纹检测
  - 设备指纹生成和验证
  - 检测设备是否为新设备
  - 检测设备是否在黑名单

- [ ] **DeviceRiskScoreComponent** - 设备风险评分
  - 基于设备历史行为评分
  - 检测设备关联的用户数量（一机多号）

#### 2.1.3 用户相关组件

- [ ] **UserBlacklistCheckComponent** - 用户黑名单检测
  - 检查用户ID是否在黑名单
  - 支持多维度黑名单（用户ID、手机号、邮箱）

- [ ] **UserProfileComponent** - 用户画像加载
  - 加载用户历史风险等级
  - 加载用户行为特征
  - 计算用户信用分

### 2.2 完善规则执行组件

**目标**：提供更强大的规则引擎能力

#### 2.2.1 频率检测组件

- [ ] **FrequencyCheckComponent** - 频率检测
  - 检测单位时间内的操作频率
  - 支持滑动窗口统计
  - 示例：1分钟内登录失败超过5次

#### 2.2.2 速度检测组件

- [ ] **VelocityCheckComponent** - 速度检测
  - 检测操作速度异常
  - 示例：1小时内从北京和上海两地登录

#### 2.2.3 决策表组件

- [ ] **DecisionTableComponent** - 决策表
  - 支持多条件组合决策
  - 可视化配置决策表

#### 2.2.4 评分卡组件

- [ ] **ScoreCardComponent** - 评分卡
  - 支持多维度评分
  - 加权计算总分

### 2.3 数据库集成

**目标**：从数据库加载配置，支持动态更新

- [ ] **黑名单服务完善**
  - ✅ 已实现基础功能
  - [ ] 添加缓存刷新接口
  - [ ] 支持黑名单过期自动清理

- [ ] **规则服务完善**
  - ✅ 已实现基础功能
  - [ ] 添加规则热更新接口
  - [ ] 支持规则版本管理

- [ ] **流程配置加载**
  - [ ] 从数据库加载流程定义
  - [ ] 支持流程动态切换

### 2.4 决策日志完善

**目标**：完整记录决策过程，便于审计和分析

- [ ] **日志记录增强**
  - ✅ 已实现基础日志记录
  - [ ] 记录每个节点的执行详情
  - [ ] 记录规则命中详情
  - [ ] 记录 AI 分析详情

- [ ] **日志查询接口**
  - [ ] 按事件ID查询
  - [ ] 按用户ID查询
  - [ ] 按时间范围查询
  - [ ] 按决策结果查询

### 3.2 规则引擎增强

**目标**：支持更复杂的规则表达式

#### 3.2.1 集成 Aviator 表达式引擎

- [ ] **Aviator 集成**
  - 替换简单的表达式解析
  - 支持复杂的逻辑运算
  - 支持自定义函数

示例：
```java
// 当前简单实现
"isNewDevice == true"

// Aviator 增强后
"isNewDevice == true && (loginHour < 6 || loginHour > 22) && failedLoginCount >= 3"
```

#### 3.2.2 规则组合

- [ ] **规则集（RuleSet）**
  - 多条规则组合成规则集
  - 支持 AND/OR 逻辑

#### 3.2.3 规则测试

- [ ] **规则测试工具**

## 9. 风险和挑战

### 9.1 技术风险

- **LiteFlow 版本升级**：关注 LiteFlow 版本更新，及时升级
- **性能瓶颈**：大规模并发下的性能优化
- **数据一致性**：分布式环境下的数据一致性保证

### 9.2 业务风险

- **规则误判**：规则配置不当导致误判
- **AI 成本**：LLM API 调用成本控制
- **数据安全**：敏感数据泄露风险

### 9.3 应对措施

- **充分测试**：单元测试、集成测试、压力测试
- **灰度发布**：新功能灰度上线，逐步放量
- **监控告警**：完善监控体系，及时发现问题
- **应急预案**：制定应急预案，快速回滚
